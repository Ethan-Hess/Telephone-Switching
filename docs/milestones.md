# Project Milestones & Testing Plan

## Overview

This project requires a comprehensive test plan outlining the strategy and scope of testing activities to ensure the project meets its requirements and quality standards.

The project is structured in **three milestones**:

1. **Milestone One** - Code development and test plan creation
2. **Milestone Two** - Testing execution
3. **Milestone Three** - Final testing and documentation

---

## Milestone One - Part 1

### Requirements

Milestone One deliverables:

- [ ] Source Code
- [ ] Test Plan
- [ ] Working executable module
- [ ] List of test cases (with execution status)

**Note:** Testing will be completed in Milestones Two and Three.

---

## Deliverables & Grading

### Special Note

Prepare a package for the assigned tester by **one day after the due date**. The instructor will identify the team or individual responsible for testing your project.

### Grading Criteria

**Program Quality (Critical)**

- The program must work correctly
- This demonstrates the quality of your testing approach

**Test Plan Quality**

- Appropriate approach for the problem
- Well-thought-out and complete strategy
- No obvious areas missed

**Test Case Completeness**

- Comprehensive coverage
- All major functionality addressed
- Edge cases considered
